# Claims And Limitations

This repository is an honest portfolio simulation. Every reviewer-facing claim should map to an executable script, local artifact, static workbook, or explicit design artifact.

## Allowed Claims

| Claim | Evidence |
| --- | --- |
| Simulated CX workbook and dashboard | `index.html`, `cx-data.js`, `dashboard-workbook/customer_experience_dashboard.xlsx` |
| Deterministic telemetry simulation | `industrial-cx-ai-lab/src/telemetry_generator.py`, `industrial-cx-ai-lab/data/simulated_telemetry.csv` |
| Controlled anomaly injection | `industrial-cx-ai-lab/src/telemetry_generator.py`, `industrial-cx-ai-lab/artifacts/anomaly_metrics.json` |
| Rule-based and IsolationForest anomaly baseline | `industrial-cx-ai-lab/src/anomaly_detection.py` |
| Service-risk classifier | `industrial-cx-ai-lab/src/service_risk_model.py`, `industrial-cx-ai-lab/artifacts/service_risk_metrics.json` |
| OPC UA / MQTT / Influx / Grafana-style artifacts | `industrial-cx-ai-lab/ops/`, `industrial-cx-ai-lab/artifacts/influx_line_protocol.txt` |
| FastAPI-compatible local scoring | `production-sim-stack/src/api.py`, `production-sim-stack/src/model_adapter.py` |
| Slurm-ready packaging | `hpc/run_industrial_scoring_array.sbatch` |
| Local CPU benchmark | `benchmarks/industrial_scoring_benchmark.py`, `benchmarks/artifacts/local_cpu_benchmark.json` |

## Do Not Claim

- No access to any company's systems, customer data, dealer data, telemetry, dashboards, or internal processes.
- No real OPC UA, Kepware, MQTT, InfluxDB, Grafana, Telegraf, Slurm, cloud, edge, or production integration.
- No real employer, partner, client, cluster or production execution.
- No production predictive-maintenance model, warranty model, safety model, pricing model, or automated customer-treatment system.
- No validated model quality beyond local metrics on synthetic data.

## Remaining Technical Limits

- Data is synthetic and intentionally small enough for CPU-friendly review.
- IsolationForest is a local benchmark against synthetic labels, not validated plant anomaly detection.
- Service-risk labels are generated by simulation rules.
- Docker Compose is optional and not required for default tests.
- Slurm scripts are packaging examples and were not submitted to a real cluster.
