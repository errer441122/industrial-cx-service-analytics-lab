# Industrial CX & Service Analytics Lab

Customer experience and service analytics case study built with simulated data.

This project shows how I would structure a junior analytics workflow:
- define customer/service KPIs;
- segment customer signals;
- build a dashboard workbook;
- identify bottlenecks and risk signals;
- translate findings into clear improvement actions;
- document assumptions and limitations.

Best fit:
- CX Analyst Intern
- Marketing Analyst Intern
- Digital Analytics Intern
- Trade Marketing Intern
- Market Intelligence Intern

## What this project is for

This repository is positioned for marketing, CX and digital analytics screening. It focuses on business-facing analysis rather than industrial AI engineering.

## Recruiter route

1. Read this README for the project positioning.
2. Review the KPI logic and segmentation framing in the parent portfolio.
3. Use the technical appendix only if the role also values Python, SQL, validation or reproducible testing evidence.

## Boundaries

This is a portfolio case study using simulated data only. It does not claim access to real company, client, customer, dealer, telemetry, advertising, CRM or service data.

This repository also includes optional technical evidence for Python validation, SQL views, industrial AI simulation and reproducible testing. These are supporting materials, not the main positioning for marketing/CX analytics roles.
